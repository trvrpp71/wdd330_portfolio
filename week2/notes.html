Chapter 2 Variables

biggest change is rather than var, the words "const" and "let" are used.
- const is for a constant that will never change
- let is to assign temp variables

const and let are considered block scoped and only exist inside the block in which they exist

if a variable is defined outside a block, then it will be global

Template Literals: introduced in ES6 and use the ` character. It allows for a couple things:
- use ' or " in the string
- allow for a JS expression to be inserted inside the string; preface it with a ${xxx}. For example
const name = 'Siri'
`hello ${name})!`

Symbols: also new in ES6; used to create unique values which helps to avoid naming collisions

Chapter 4 Functions

Three of the most common way of defining a function:

1. Function Declarations

simply the function keyword and the name of the function in parentheses.

2. Function Expressions

This assigns a anonymous or named function to a variable.

3. Function Constructors

This is where the body of the function is entered as a string.

NOTE: this way is not recommended due to numerous problems that arise when doing this.

4. New way with ES6 - Arrow

identified by the arrow symbol =>
parameters come before the arrow
main body of the function after
ex. const square = x => x*x

Parameters and Arguements

parameters are values that the function needs and are determined BEFORE the function runs

arguments are the values passed to the function to do something with


Callbacks

when a function passes a result to another calling function
